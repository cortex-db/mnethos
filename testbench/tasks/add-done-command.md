Add a `done` command to this todo CLI.

`python todo.py done <id>` should mark the todo with the given id as completed
(set its `done` field to `true`), print a short confirmation, and persist the
change. If no todo with that id exists, print an error message and exit with a
non-zero status.

Follow the project's existing conventions and make sure the full test suite
passes before you finish.
